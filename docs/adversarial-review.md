# 对抗性审查

## 已处理

- Loading 页不增加需求文档未确认的营销正文。
- 题目页采用“选择后自动进入下一题”，避免新增未确认的“下一题/查看结果”按钮文案。
- UI 参考图只用于红黑电竞视觉、斜切按钮、发光框体、声波和转盘风格，不覆盖正式文案。
- KV 只作为风格和字体样式参考，不直接裁切或用于页面背景。
- H5 画面和生成海报不直接使用白门产品图作为画面元素，改用霓虹门框、空间线稿和扫描轮廓表达静音门概念。
- 音乐音效使用 Web Audio 原创生成，不引入外部版权音乐。
- 门体大图生成 WebP 版本，避免移动端加载几十兆原图。
- 抽奖结果由 mock API 返回，前端不自行暴露概率和库存。

## 仍需活动方确认

- 活动开始和结束时间。
- 正式域名、测试域名和 GitHub Pages 是否只作为预览。
- 微信公众号/AppID 与 JS-SDK 签名接口。
- 奖券总量、奖项名称、数量和概率。
- 无库存或未中奖正式文案。
- 隐私政策正式链接。
- 城市列表和城市 ADMIN 账号。
- 是否需要短信验证、门店字段和正式核销规则。
